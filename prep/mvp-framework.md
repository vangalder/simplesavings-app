# MVP Framework & Product Constraints

> This framework forms the basis for a strong Minimum Viable Product (MVP) and serves as the primary constraint for all feature development.

## 1. Core Function
The one essential thing the app does. If you stripped away everything else, this is what remains.
- **YouTube:** Video playback
- **Google:** Search
- **boat.bot:** instant AI boat mechanic via voice
- **cuentacuentos.es:** Customized storytelling

## 2. Core Loop (Feedback Loop)
Turn the core function into a quick, rewarding action-reward cycle (ideally under ~30 seconds). 
- **Input Ease:** The effort to start the loop must be near zero.
- **Immediate Reward:** Use visuals, haptics, or progress indicators.

## 3. Accessory Features & The "Kill List"
Supporting elements that enhance the core function (logs, charts, customization).
- **The Constraint:** To prevent bloat, every new accessory should ideally justify the removal of an underperforming one.

## 4. Surface Area Check & Friction Audit
Keep the app intuitive so users understand it in one run-through.
- **Friction Audit:** Can a user experience the Core Function without an account? 
- **Onboarding:** Minimal friction; maximize "Time to Value" (TTV).

## 5. Retention Hook: Variable Rewards
Avoid "notification debt." Instead of reminders, provide **Variable Rewards**.
- **The Hook:** What is new or different every time they return?
- **State:** Create a reason to return that feels like a discovery, not a chore.

## 6. Monetization: Contextually Curative
Affiliate income should be a "trickle" that feels like a service.
- **Strategy:** The link must solve a problem the Core Loop just identified.
- **Trust:** Monetization should feel like a "Level Up," not a toll booth.
